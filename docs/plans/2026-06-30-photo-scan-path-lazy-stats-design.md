# 照片页扫描路径统计延迟加载设计

## 背景

照片管理页首次加载时会读取扫描路径，并立即请求 `/photos/derived-status-by-paths`。该派生统计会扫描全部活跃照片；NAS 实测约需 1.5–2 秒，并与照片列表总数查询争抢 SQLite 读连接和磁盘 I/O。

扫描路径区域的折叠状态已保存在 `localStorage`。折叠时用户看不到每条路径的照片、缩略图和地理编码统计，因此无需在首屏计算这些数据。

## 设计

- 扫描路径配置仍在首屏加载，用于显示路径数量、识别正在运行的扫描任务，以及在展开时立即渲染路径列表。
- 仅当扫描路径区域处于展开状态时请求派生统计。
- 用户从折叠切换到展开时，如果尚未加载派生统计，则请求一次。
- 已加载的派生统计在本次页面生命周期内复用；重复折叠和展开不重复请求。
- 原有扫描完成、重建完成和批量操作后的显式刷新逻辑保持不变。

## 验证

- 单元测试覆盖：折叠状态不加载、展开状态加载、已经加载后不重复加载。
- 运行前端 TypeScript 编译与 Vite 生产构建。
